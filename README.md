Проєкт являє собою програмний модуль інформаційної системи «Електронна медична карта», побудований на принципах модульної архітектури 
для забезпечення високої масштабованості та гнучкості системи.
Процес розробки базувався на попередньому функціональному моделюванні за стандартом DFD, що дозволило детально описати вхідні потоки даних, 
механізми управління та взаємодію основних бізнес-процесів медичної установи.
Проектування системи реалізовано за допомогою UML-діаграм, що забезпечило чітке структурування бази даних (Class Diagrams) та логічну послідовність 
взаємодії об'єктів.
Програмна реалізація виконана на фреймворку Django з розподілом логіки на окремі спеціалізовані додатки, що сприяє дотриманню принципів фреймворку.

### Технологічний стек
* **Backend:** Python 3.12, Django 5.2 (MTV architecture)
* **Database:** PostgreSQL
* **Frontend:** HTML, CSS, Bootstrap5
* **Modeling:** UML (Use Case, Class diagrams), DFD


### Структура проєкту
Проєкт розділений на модулі (Django Apps) для забезпечення масштабованості:
* users_app — система автентифікації та профілі користувачів.
* cards_app — ядро системи: управління електронними медичними картами.
* laboratory_app — модуль для обліку та перегляду результатів аналізів.
* public_app — інформаційна частина та лендинг системи.
* core — містить логіку, що виконується всією системою для захисту - хешування ID url-адресах
* messages_app(в планах) — модуль для розсилки системних повідомлень за шаблонами.


### Проектування системи
В папці **docs** Ви знайдете архітектурні діаграми, розроблені під час побудови архітектури проекту:
* Діаграма прецедентів: сценарії взаємодії користувачів із системою.
* Діаграма класів: структура даних та зв'язки між сутностями ЕМК.
* IDEF3: декомпозиція процесів управління доступом.
